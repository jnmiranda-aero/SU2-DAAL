int CRadialBasisFunction::CheckPolynomialTerms ( su2double  max_diff_tol,
vector< int > &  keep_row,
su2passivematrix &  P 
)
static

If the polynomial term is included in the interpolation, and the points lie on a plane, the matrix becomes rank deficient and cannot be inverted. This method detects that condition and corrects it by removing a row from P (the polynomial part of the interpolation matrix).

Parameters
[in]max_diff_tol- Tolerance to detect whether points are on a plane.
[out]keep_row- Marks the dimensions of P kept.
[in,out]P- Polynomial part of the interpolation matrix, one row may be eliminated.
Returns
n_polynomial - Size of the polynomial part on exit (in practice nDim or nDim-1).

void CRadialBasisFunction::ComputeGeneratorMatrix ( RADIAL_BASIS  type,
bool  usePolynomial,
su2double  radius,
const su2activematrix &  coords,
int &  nPolynomial,
vector< int > &  keepPolynomialRow,
su2passivematrix &  C_inv_trunc 
)
static

Compute the RBF "generator" matrix with or without polynomial terms.

Note
Multiplying C_inv_trunc by a column vector gives specific coefficients for given "known values", conversely, multiplying (on the left) by a row vector of polynomial and RBF values gives generic interpolation coefficients for a given target evaluation point.
Parameters
[in]type- Type of radial basis function.
[in]usePolynomial- Whether to use polynomial terms.
[in]radius- Normalizes point-to-point distance when computing RBF values.
[in]coords- Coordinates of the donor points.
[out]nPolynomial- Num of poly terms, -1 if !usePolynomial, nDim-1 if coords lie on plane, else nDim.
[out]keepPolynomialRow- Size nDim, signals which (if any) iDim was removed from polynomial term.
[out]C_inv_trunc- The generator matrix as described above.
