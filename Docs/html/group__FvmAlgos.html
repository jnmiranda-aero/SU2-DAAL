Compute the gradient of a field using the Green-Gauss theorem.

Note
Template nDim to allow efficient unrolling of inner loops.
Gradients can be computed only for a contiguous range of variables, defined by [varBegin, varEnd[ (e.g. 0,1 computes the gradient of the 1st variable). This can be used, for example, to compute only velocity gradients.
The function uses an optional solver object to perform communications, if none (nullptr) is provided the function does not fail (the objective of this is to improve test-ability).
Parameters
[in]solver- Optional, solver associated with the field (used only for MPI).
[in]kindMpiComm- Type of MPI communication required.
[in]kindPeriodicComm- Type of periodic communication required.
[in]geometry- Geometric grid properties.
[in]config- Configuration of the problem, used to identify types of boundaries.
[in]field- Generic object implementing operator (iPoint, iVar).
[in]varBegin- Index of first variable for which to compute the gradient.
[in]varEnd- Index of last variable for which to compute the gradient.
[in]idxVel- Index of velocity, or -1 if no velocity present.
[out]gradient- Generic object implementing operator (iPoint, iVar, iDim).

Generic limiter computation for methods based on one limiter value per point (as opposed to one per edge) and per variable.

Note
This implementation can be used to derive most common methods by specializing the limiter functions (e.g. Venkatakrishnan) and the geometric modifications (e.g. sharp edges), this is done via specialization of "CLimiterDetails" (all its methods). Then, a call to this function with the specialized "LimiterKind" should be added to the body of "computeLimiters()". See also the notes in computeGradientsGreenGauss.hpp

Arguments:

Parameters
[in]solver- Optional, solver associated with the field (used only for MPI).
[in]kindMpiComm- Type of MPI communication required.
[in]kindPeriodicComm1- Type of periodic comm. to determine min/max.
[in]kindPeriodicComm2- Type of periodic comm. to adjust limiters.
[in]geometry- Geometric grid properties.
[in]config- Configuration of the problem.
[in]varBegin- First variable index for which to compute limiters.
[in]varEnd- End of computation range (nVar = end-begin).
[in]field- Variable field.
[in]gradient- Gradient of the field.
[out]fieldMin- Minimum field values over direct neighbors of each point.
[out]fieldMax- As above but maximum values.
[out]limiter- Reconstruction limiter for the field.

Template parameters:

Parameters
nDim- Number of dimensions.
LimiterKind- Used to instantiate the right details class.
FieldType- Generic object with operator (iPoint,iVar).
GradientType- Generic object with operator (iPoint,iVar,iDim).

Correct gradients on symmetry and Euler (slip) markers to respect the conditions:

  1. n.grad(phi) = 0
  2. n.grad(v.t) = 0
  3. t.grad(v.n) = 0
    Note
