◆ CSYSVEC_PARFOR

#define CSYSVEC_PARFOR   SU2_OMP_SIMD

OpenMP worksharing construct used in CSysVector for loops.

Note
The loop will only run in parallel if methods are called from a parallel region (if not the results will still be correct). Static schedule to reduce overhead, chunk size determined at initialization. "nowait" clause is safe when calling CSysVector methods after each other as the loop size is the same. Methods of other classes that operate on a CSysVector and do not have the same work scheduling must use a SU2_OMP_BARRIER before using the vector.
